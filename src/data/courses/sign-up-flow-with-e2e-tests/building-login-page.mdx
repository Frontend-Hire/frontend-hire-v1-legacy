## Tip: ESLint react/no-unescaped-entities issue

This is quite an irritating issue that you might face when using the `react/no-unescaped-entities` rule in ESLint. The rule is enabled by default in the Next.js ESLint configuration. The rule throws an error when you use special characters like `&`, `<`, `>`, etc., directly in JSX. To fix this issue, you can use the `&amp;`, `&lt;`, `&gt;`, etc., HTML entities instead of the special characters.
